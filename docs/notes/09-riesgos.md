# Guion — Gestión de riesgos

## Contexto para el presentador

Esta diapositiva muestra la matriz de riesgos. El objetivo es demostrar que el equipo identificó y planificó contingencias.

## Qué decir

"Identificamos tres riesgos principales y una mitigación concreta para cada uno.

El riesgo de desalineación entre backend y frontend lo mitigamos con una matriz de estado funcional que revisamos en cada hito.

La vulnerabilidad en autenticación se previene con validación estricta de entorno y rotación periódica de credenciales.

La degradación bajo concurrencia se aborda con las pruebas de carga y stress que ya implementamos.

Además de estos, también tenemos cubiertos los errores en flujos de pago y webhook con las pruebas de integración, y la integridad del repositorio con PR obligatorio y checks de CI."

## Tiempo estimado

1–2 minutos
